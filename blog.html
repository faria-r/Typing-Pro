Tha data we store in Local Storage is permanent.It doesn't gets cleared automatically when the page is reload or closed the session until we remove it.We can apply multiple method for storing, removing and gettting data from local Storage.Such as:
localStorage.getItem(key)
localStorage.setItem(key,value)
localStorage.removeItem(key)
localStorage.clear()
The data stored into session storage is temporay and it gets clear if we closed the session or reload the page. It also has different methods like localStorage.They are:
sessionStorage.getItem(key)
sessionStorage.setItem(key,value)
sessionStorage.removeItem(key)
sessionStorage.clear()

Global Scope means we can access the variable or any object or array from any where in a JavaScript program if it declared globally(not inside any funciton or loop).
Block scope means anything declared within a block can not access outside of that block.
But the variable declared with var can not have block Scope.

The event loop is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks. If there are any, it pushes it to the Call Stack, waits for the Call Stack to be clear again, and repeats the same process.

There are eight ways How we get undefined:
1.variable that is not initialized will give undefined.
2.function with no return will give undefined.
3.parameter that is not passed while function calling will give undefined
4.If return has nothing on the right side it will give undefined.
5.Trying to access a property that doesn't exist in a object will give undefined.
6.Accessing array elements outsides of index range
7.Deleting an element inside of a array.
8.Set a value directly to undefined
